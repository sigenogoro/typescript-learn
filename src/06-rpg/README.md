# 06 - CLI RPG（学習の集大成）

01〜05 で学んだ TypeScript の知識をすべて活用して、コマンドラインで動く RPG を作る。

## ゲームの仕様

# CLI RPGバトルシステム 仕様書

## 1. 概要
CLI上で動作する、1対1のターン制RPGバトルシステムを作成する。
プレイヤーと敵キャラクターが交互に行動し、どちらかのHPが0になるまで戦闘を続ける。

## 2. バトルの基本ルール
- バトルはプレイヤー1体、敵1体で行う
- 各キャラクターはHP、攻撃力、防御力、素早さなどの能力値を持つ
- 毎ターン、プレイヤーは行動を選択する
- 敵は自動で行動を選択する
- HPが0以下になったキャラクターは戦闘不能になる
- どちらかが戦闘不能になった時点でバトル終了

## 3. CLIでの操作
プレイヤーのターンでは、以下のような選択肢を表示する。

1. 攻撃
2. 防御
3. 回復
4. スキル

プレイヤーは番号を入力して行動を決定する。

## 4. キャラクターの項目
キャラクターは以下の情報を持つ。

- 名前
- 最大HP
- 現在HP
- 攻撃力
- 防御力
- 素早さ
- 属性
- 状態異常
- 使用できるスキル一覧

## 5. 属性
属性によってダメージ倍率が変化する。

例：

- 火は草に強い
- 草は水に強い
- 水は火に強い

倍率例：

- 有利：1.5倍
- 不利：0.5倍
- 通常：1.0倍

## 6. 行動の種類

### 攻撃
通常攻撃を行い、相手にダメージを与える。

### 防御
そのターンに受けるダメージを軽減する。

### 回復
自分のHPを回復する。

### スキル
特殊な攻撃や状態異常を発生させる。

## 7. ダメージ計算
ダメージは以下の要素で決まる。

- 攻撃側の攻撃力
- 防御側の防御力
- スキル威力
- 属性相性
- 防御中かどうか
- ランダム補正

例：

ダメージ = 攻撃力 + スキル威力 - 防御力

その後、属性倍率や防御倍率をかける。

## 8. 状態異常
状態異常によって、キャラクターの行動や能力に影響を与える。

例：

- 毒：毎ターン終了時にHPが減る
- 麻痺：一定確率で行動できない
- 攻撃力低下：攻撃力が一時的に下がる
- 防御力低下：防御力が一時的に下がる

## 9. ターンの流れ
1. プレイヤーと敵の素早さを比較する
2. 素早い方から行動する
3. 行動結果をCLIに表示する
4. HPが0になったか確認する
5. 状態異常の効果を処理する
6. 次のターンへ進む

## 10. 勝敗条件
- 敵のHPが0になったらプレイヤーの勝利
- プレイヤーのHPが0になったら敗北

## 11. 表示する戦闘ログ例

1. 攻撃
2. 防御
3. 回復
4. スキル

勇者の攻撃！

スライムの攻撃！

勇者は毒で5ダメージを受けた！
