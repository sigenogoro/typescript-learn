# 04 - ジェネリクス

「型をパラメータとして受け取る」仕組みを学ぶ。コードを再利用しながら型安全を保つための核心。

## 学習内容

- **ジェネリック関数** — `function identity<T>(arg: T): T`
- **ジェネリッククラス** — クラスに型パラメータを持たせる
- **制約（Constraints）** — `extends` で型パラメータを絞り込む
- **複数の型パラメータ** — `<T, U>` のように複数定義する
- **デフォルト型パラメータ** — `<T = string>` で省略時のデフォルトを設定する

## 学習のポイント

- `T` は慣習的な名前（`Type` の略）。`K`, `V`, `U` なども慣習的に使われる
- ジェネリクスを使うことで `any` を使わずに汎用的なコードが書ける
- `<T extends object>` のように制約を付けることで、`T` に使えるプロパティを保証できる
- React や NestJS では至るところでジェネリクスが使われている（例: `Promise<T>`, `Observable<T>`）
